# cannot-reproduce（仮）

## エレベーターピッチ

運用に疲れたエンジニアが、**30秒**でどれだけ「定時退社」に近づけるかを競う、瞬間判断のスコアアタック。通知されるバグ報告の**報告者の肩書き/名前**から「重要人物」を見抜き、

* 重要人物 → **Active（対応中）**
* その他大勢 → **再現待ち完了**
  で即振り分ける。連続正解でタイムが伸び、ミスで減る。社会の“忖度”と現場の“処世術”をゲーム化。

---

## 世界観 / トーン

* プレイヤーは「**運用にうんざりしているエンジニア**」。目標は**ただ一つ：定時退社**。
* 予算を握る顧客、上司の上司、密かに想いを寄せる社内のあの子——**この3カテゴリは最優先**。
* それ以外は\*\*全部“再現待ち完了”\*\*で片付けたい…が、たまに重要人物ラベルがまぎれてくるので要注意。
* トーンは**風刺×ユーモア**。一般化・比喩で表現し、個人や組織を特定しない。

---

## コアゲームループ（1プレイ=30秒）

1. 画面上部から**バグ報告カード**が次々出現（1～1.5秒間隔、難易度で変動）。
2. 各カードには **報告者名/肩書き/所属** と **件名・要約**、**緊急度アイコン** が記載。
3. プレイヤーは**左ボタン/キー**で「再現待ち完了」、**右ボタン/キー**で「Active（対応中）」を即時入力。
4. **正解**：+1.0～+1.5秒（コンボで加算幅UP） / **ミス**：-2.0秒（コンボリセット）。
5. 残り時間が0になると終了。ハイスコアを更新。

---

## 正解判定ロジック（忖度システム）

* ゲーム開始時に\*\*ランダムで「重要人物シード」\*\*を生成：

  * ① 予算を握る顧客（例：案件PM、経営層、キーパーソン）
  * ② 上司の上司（例：部長、事業部長、CTO）
  * ③ 社内の気になる人（例：同部署/他部署のメンバー）
* これら**3カテゴリの報告だけ**「Active」が正解。それ以外は常に「再現待ち完了」が正解。
* シードは**名前/肩書き/所属/メールドメイン/アイコン色**など複数の手掛かりで示唆。
* ラウンド中に**重要人物が入れ替わるイベント**（高難度）：画面に小さく合図（例：社内ポータルの人事ニュース風トースト）。

---

## 操作

* **PC**：Left ← / A = 再現待ち完了、 Right → / D = Active、 Space = 一時停止、 Enter = スキップ（1プレイ1回）
* **モバイル**：画面左タップ = 再現待ち完了、右タップ = Active、上スワイプ = スキップ
* **アクセシビリティ**：キーリマップ、長押し無効（誤入力防止）、触覚フィードバック（モバイル）

---

## タイム＆スコア

* **初期タイム**：30.0秒
* **正解**：+1.0秒（コンボ10/20/30で +0.2 / +0.3 / +0.5 まで上昇）
* **ミス**：-2.0秒 & コンボ0
* **連続正解ボーナス**：10/20/30で**判定受付時間**を+50/+75/+100ms緩和（手触り向上）
* **フィーバー**：コンボ30到達で5秒間、正解+1.5秒・ミス-1.0秒。

---

## 難易度設計

* **Easy**：出現間隔1.5s、誤誘導少、重要人物は明確な肩書き表示
* **Normal**：1.2s、似た役職名で混乱、偽ドメイン（.co vs .com）
* **Hard**：0.9s、アイコン・敬称・所属略称でミスリード、**重要人物の入れ替わり**あり
* **Insane**：0.7s、重要人物が同時2名、件名・本文が皮肉/社内用語で判別困難

## カード生成（例）

**メタデータ**：

* `sender_name`（例：佐伯 由紀）
* `role`（例：顧客PM / 部長 / 経理 / アシスタント）
* `org`（例：御曹司ホールディングス / 第3開発部）
* `contact`（メール/社内チャットID）
* `urgency`（LOW/MID/HIGH/MAX）
* `subject`（例：`[MAX] 本番で白画面（Win11/Chrome）`）
* `body`（1～2行）

**判別の手掛かり（例）**：

* 重要人物の**肩書きアイコン**がわずかに光る（視認性は低め）
* メールドメインが`@keyclient.co`（類似 `@keycllent.co` を混ぜる）
* 社内ポータルに「昇進」のトーストが流れた直後に、その人からの報告が増える

## 実装メモ（Unity想定）

* **レンダパイプライン**：Built-in（2D UI中心）
* **シーン**：Boot / Title / Game / Result
* **データ**：ScriptableObjectで名前・肩書き・所属のプール、重み付きランダム
* **生成**：Object Poolingでカード再利用
* **入力**：New Input System。PC/モバイル両対応
* **時限**：`Time.deltaTime`で残り時間管理、判定受付のグレース期間をDifficultyで調整
* **保存**：PlayerPrefs or JSON（将来オンラインLBならバックエンド連携）

## アーキテクチャとフォルダ構成（クリーンアーキテクチャ）

開発の保守性・拡張性を高めるため、クリーンアーキテクチャの原則に基づいた構成を採用します。依存性のルール（内側は外側を知らない）を徹底し、ゲームのコアロジックとUnityフレームワークへの依存を分離します。

### フォルダ構成案 (`Assets/CannotReproduce/`)

```
Assets/
└── CannotReproduce/
    ├───Scripts/
    │   ├───Domain/         # コアロジック (Unity非依存)
    │   │   ├───Entities/      # ゲームのデータ構造 (Score, Timer, CardData, etc.)
    │   │   └───UseCases/     # ゲームのルール・シナリオ (SortCardUseCase, StartGameUseCase, etc.)
    │   ├───Infrastructure/ # データ永続化、外部サービス連携
    │   │   ├───Repositories/ # ハイスコア保存など (PlayerPrefs, JSON)
    │   │   └───DataSources/  # ScriptableObjectからのデータ読み込み
    │   └───Presentation/   # UIと入出力 (Unity依存)
    │       ├───Controllers/  # ユーザー入力の受付 (Input System -> UseCase)
    │       ├───Presenters/   # UseCaseの結果をUIに反映
    │       └───Views/        # UIコンポーネント (MonoBehaviour)
    ├───Data/             # ScriptableObjectアセット
    │   ├───CardTemplates/
    │   └───GameSettings/
    ├───Prefabs/          # 再利用可能なオブジェクト
    ├───Scenes/           # ゲームシーン
    ├───Input/            # Input Actions アセット
    └───(Audio, Sprites, etc.)
```

### 依存関係のルール

*   **Presentation -> UseCases**: ユーザー入力（`Controllers`）をきっかけに`UseCases`を呼び出す。`Presenters`が`UseCases`の実行結果を監視し、`Views`を更新する。
*   **Infrastructure -> Domain**: `Infrastructure`層は`Domain`層で定義されたインターフェース（例: `IRepository`）を実装する。
*   **Domain**: 他のレイヤーに依存しない、純粋なC#コードで記述する。

### DI (Dependency Injection)

各レイヤー間の依存関係を疎結合に保つため、VContainerやZenjectのようなDIコンテナの導入を推奨します。これにより、テスト容易性の向上も期待できます。

## リスクと緩和

* **倫理**：忖度の風刺が“助長”に見えないよう、**最後に皮肉の一文**（例：『現実では、すべての報告に敬意を』）をリザルトに表示。
* **可読性**：氏名/役職が読みにくいとゲームにならない→最小フォント/行間ルールを厳守
* **学習コスト**：2ボタン操作で最小化。最初の5件は**明瞭な教材カード**を流す

---

## MVP範囲（2～3日目安の最小）

* 2ボタン判定・30秒タイマー・コンボ/加減秒・3種カードテンプレ・Easyのみ
* Resultでスコア/最大コンボ/重要人物名を表示
* ローカルランキング

## 用語集

* **再現待ち完了**：ここでは“対応を後回しにする”皮肉的ラベルのこと
* **重要人物**：ゲーム内カテゴリ名。実在の個人/組織は登場しない
* **フィーバー**：高コンボ時の一時的救済＆爽快演出
